# Echo — Feature: Review Learning Loop and Same-Day Extra Study UX

## Context

Текущий review-flow слишком рано считает карточку “пройденной” внутри сессии:

- после `Again` или `Hard` карточка исчезает из текущей review-session;
- пользователь может закончить сессию, даже если часть слов он всё ещё не знает;
- сразу после этого интерфейс может предлагать extra study, хотя пользователь уже занимался сегодня.

Это расходится с ожидаемой учебной логикой и воспринимается как баг.

## Goal

Сделать review-session ближе к ожидаемому learning loop:

- трудные карточки должны возвращаться в эту же сессию;
- session считается завершённой только когда текущая очередь реально пройдена;
- после занятий сегодня интерфейс не должен агрессивно предлагать “новые слова”, но extra study должен оставаться доступным как добровольный режим.

## Anki-Inspired Behavior

Поведение должно быть вдохновлено логикой Anki learning/relearning:

- `Again` возвращает карточку в учебный цикл;
- `Hard` тоже не выпускает карточку из активной learning loop;
- `Good` и `Easy` позволяют двигаться дальше и завершать сессию.

В этом проекте допустима упрощённая in-session approximation:

- не нужно в точности копировать minute-based step scheduler Anki;
- важно сохранить основной UX-смысл: пока пользователь не дошёл хотя бы до “нормально”, карточка не должна исчезать из текущего учебного процесса.
- после полной дневной сессии dashboard не должен снова поднимать основной CTA `Учить` из-за карточек, которые уже review-ились сегодня.

## Queue Requirements

### 1. Requeue Rules

Для любой review-session:

- `Again` возвращает текущую карточку обратно в очередь ближе к началу;
- `Hard` возвращает текущую карточку обратно в очередь немного позже, чем `Again`;
- `Good` и `Easy` не реинсертят карточку обратно в текущую очередь.
- если карточка уже review-илась сегодня, она не должна повторно участвовать в dashboard CTA count для “Учить”.

### 2. Shared Behavior Across Session Types

Одинаковая логика должна работать для:

- обычного due review;
- manual review из фильтров на странице колоды;
- extra study.

Session type не должен менять поведение learning loop.

### 3. Session Completion

Done-screen должен показываться только когда:

- текущая очередь реально опустела;
- все requeued cards тоже закончились;
- pending review submissions уже не блокируют финальный screen state.

## Same-Day Extra Study UX

Если пользователь уже занимался сегодня и в колоде нет due cards:

- интерфейс не должен активно предлагать “новые слова” как основной CTA;
- extra study должен оставаться доступным только как осознанный дополнительный выбор.

Практически это означает:

- более нейтральный текст;
- отсутствие агрессивного prompt-а “Готовы к новым словам?” после уже состоявшейся учёбы;
- `+10 / +20` остаются доступными, но не навязываются.

## Storybook Requirement

Если extra-study box визуально меняет tone/state в same-day сценарии, это должно быть отражено в Storybook.

## Tests Requirement

Минимально нужны automated tests на:

- requeue logic для `Again / Hard / Good / Easy`;
- одинаковое поведение queue helper для любых review-session inputs;
- same-day extra study presentation helper, если он вынесен в чистую функцию.

## Acceptance Criteria

- [ ] `Again` возвращает карточку обратно в текущую review-session.
- [ ] `Hard` тоже возвращает карточку обратно в текущую review-session.
- [ ] `Good` и `Easy` выпускают карточку из текущей review-session.
- [ ] После review сегодня карточка не поднимает dashboard CTA `Учить` повторно в тот же день.
- [ ] Обычная due review и manual review используют одну и ту же learning-loop механику.
- [ ] Done-screen не показывается, пока requeued cards ещё остались в очереди.
- [ ] После занятий сегодня dashboard не предлагает “новые слова” агрессивным prompt-ом.
- [ ] Extra study остаётся доступным как добровольное действие.
- [ ] Новая queue logic покрыта тестами.
