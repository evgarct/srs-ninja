# Echo — Feature: Review Session Prefetch and Instant Progression

## Context

Во время review-session интерфейс ощущается медленным, особенно на мобильных устройствах:

- раскрытие карточки ощущается затянутым;
- после оценки следующая карточка появляется с заметной задержкой.

Это ухудшает flow повторения и делает мобильный опыт менее приятным, даже если сами данные уже доступны.

## Goal

Сделать review-session заметно более отзывчивой:

- переворот карточки должен ощущаться быстрым и лёгким;
- переход к следующей карточке должен происходить сразу после выбора рейтинга, без ожидания сети;
- нужные данные и ближайшие ассеты должны быть подготовлены заранее.

## UX Requirements

### 1. Faster Reveal

Когда пользователь раскрывает ответ:

- reveal должен происходить без заметной паузы;
- анимации не должны выглядеть “тяжёлыми” или излишне длинными;
- поведение карточки при reveal не должно меняться функционально, только по скорости и отзывчивости.

### 1.1 Touch Responsiveness

На touch-устройствах review-flow должен ощущаться ближе к нативному приложению:

- интерактивные элементы должны реагировать на касание сразу, без ощущения “только после click”;
- основные review-controls должны использовать mobile-friendly touch behavior;
- у tappable controls должен быть мгновенный pressed-feedback;
- hit area кнопок в review-flow не должна быть слишком мелкой для телефона.

### 2. Instant Next Card

Когда пользователь ставит рейтинг:

- следующая карточка должна появляться сразу;
- UI не должен ждать завершения server roundtrip перед переходом;
- статистика сессии и прогресс-бар должны обновляться немедленно.

### 3. Prepared Session Data

Перед показом карточек клиент должен заранее подготовить review-session model:

- mapping note fields -> flashcard props;
- direction (`recognition` / `production`);
- interval labels для rating buttons;
- доступный `audioUrl`.

Подготовка не должна выполняться “лениво только для текущей карточки”, если это создаёт лишнюю паузу при переходе дальше.

### 4. Lookahead Asset Prefetch

Для текущей и ближайших карточек должен быть lookahead prefetch по аудио:

- prefetch делается только для реально существующих audio URLs;
- одинаковые URLs не должны префетчиться повторно;
- prefetch не должен ломать обычное проигрывание аудио.

## Data / Sync Requirements

### Review Submission

Сохранение результата review всё ещё обязательно, но:

- запись review не должна блокировать переход к следующей карточке;
- отправка результата должна происходить в фоне;
- если в момент завершения сессии ещё идёт сохранение, UI должен это явно показывать.

### Failure Handling

Если фоновое сохранение review не удалось:

- сессия не должна зависать посреди карточек;
- пользователь должен увидеть, что результаты сохранились не полностью.

## Non-Goals

- не меняем FSRS-алгоритм;
- не меняем порядок карточек в очереди;
- не меняем логику autoplay;
- не добавляем новый режим review.

## Technical Direction

Ожидаемое направление реализации:

- выделить helper для подготовки review queue items заранее;
- переиспользовать эту подготовленную очередь внутри `ReviewSession`;
- унести submit из критического UI-path;
- добавить lookahead prefetch для аудио текущей и ближайших карточек;
- сократить слишком длинные reveal-related transition durations в `Flashcard`;
- добавить touch-oriented interaction tuning для card reveal, rating buttons и review buttons.

## Tests Requirement

Новый функционал должен быть покрыт automated tests хотя бы на уровне чистых helper-функций.

Минимально должны быть тесты на:

- подготовку review queue items из сырого card payload;
- вычисление lookahead audio URLs без дублей;
- корректное направление карточки и подстановка audioUrl.
- bypass due-card ordering для manual sessions, чтобы ручная тренировка не теряла часть `new`-карточек.

## Storybook Requirement

Если review-flow визуально меняется на touch-устройствах, это должно быть отражено в Storybook хотя бы через существующие atomic flashcard stories.

## Acceptance Criteria

- [ ] Reveal карточки ощущается быстрее, чем до фичи.
- [ ] После нажатия rating следующая карточка показывается сразу, без ожидания `submitReview()`.
- [ ] Review-session заранее подготавливает данные карточек для UI.
- [ ] Аудио для текущей и ближайших карточек префетчится lookahead-ом.
- [ ] Одинаковые audio URLs не префетчатся повторно.
- [ ] Основные review-controls на touch-устройствах дают мгновенный pressed-feedback.
- [ ] Manual review не теряет `new`-карточки из-за due-card ordering limit.
- [ ] Если в конце сессии ещё идёт сохранение review, это видно в UI.
- [ ] Если фоновое сохранение падает, пользователь видит сообщение о проблеме.
- [ ] Новая helper-логика покрыта тестами.
- [ ] Touch-oriented review changes отражены в Storybook.
