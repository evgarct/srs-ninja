# Echo — Feature: Deck Page Audio Playback and Instant Audio Refresh

## Context

На странице колоды (`/deck/[id]`) пользователь видит список нотов и может:

- понять, у каких нотов уже есть аудио;
- открыть inline-редактор ноты;
- сохранить изменения;
- при необходимости перегенерировать TTS.

Сейчас звук генерируется и хранится в `audio_cache`, но на странице колоды нет явной кнопки воспроизведения аудио в списке нотов, а UI после сохранения/регенерации опирается на серверный refresh страницы. Из-за этого новое аудио не становится доступным мгновенно внутри текущего UI-состояния.

## Goal

Добавить на странице колоды ручное воспроизведение аудио:

- в строке ноты в списке карточек;
- в режиме редактирования ноты.

После генерации или регенерации аудио:

- кнопка Play должна появляться мгновенно без перезагрузки страницы;
- только что сгенерированный URL должен сразу стать текущим источником аудио;
- новое аудио должно сразу проигрываться автоматически;
- от пользователя не требуется отдельное подтверждение или повторное нажатие.
- если генерация или запись в `audio_cache` не удалась, пользователь должен увидеть явную ошибку.

## Scope

### In scope

- Кнопка Play в каждой строке ноты на странице колоды, если у ноты есть аудио.
- Кнопка Play внутри edit-sheet для ноты, если у ноты есть аудио.
- Локальное клиентское обновление audio state после сохранения ноты и после принудительной регенерации.
- Немедленное проигрывание нового аудио после успешной генерации/регенерации.
- Синхронизация списка нотов и открытого edit-sheet без `router.refresh()` как единственного механизма для показа нового аудио.

### Out of scope

- Изменение поведения review-сессии.
- Массовая генерация аудио для всей колоды.
- Поддержка TTS для языков вне текущего списка supported languages.
- Новый пользовательский диалог подтверждения перед авто-проигрыванием свежесгенерированного аудио.

## User Stories

1. Как пользователь, я хочу нажать кнопку Play рядом с нотой на странице колоды и сразу услышать аудио для этой ноты.
2. Как пользователь, я хочу видеть такую же кнопку Play внутри edit-sheet, чтобы быстро проверить текущее произношение перед сохранением.
3. Как пользователь, после `Save Changes`, если текст ноты изменился и аудио было пересоздано, я хочу сразу услышать новое аудио и сразу видеть кнопку Play в UI.
4. Как пользователь, после `Save & Regenerate Audio`, даже если текст не менялся, я хочу сразу услышать новую версию аудио и иметь возможность сразу проиграть её повторно той же кнопкой.

## Functional Requirements

### 1. Deck page note row

Для каждой ноты на `/deck/[id]`:

- если в `audio_cache` есть `storage_path` для `note_id` и `field_key = 'expression'`, в строке ноты показывается кнопка Play;
- если аудио нет, кнопка скрыта;
- нажатие на кнопку проигрывает актуальный `audioUrl` этой ноты;
- кнопка должна быть доступна без открытия review-сессии и без перехода на другую страницу.

### 2. Edit mode playback

В `NoteEditSheet` / `NoteEditorForm`:

- если у редактируемой ноты есть аудио, показывается кнопка Play;
- нажатие на кнопку проигрывает актуальное аудио ноты;
- после успешной генерации или регенерации форма должна обновить свой локальный `audioUrl` на новый URL и использовать именно его.

### 3. Instant UI update after save

После успешного сохранения ноты:

- если `updateNoteFields(...)` вернул `audioUrl`, UI должен немедленно обновить локальное состояние ноты;
- строка ноты на странице колоды должна начать считать, что у ноты есть аудио, без ожидания полного page refresh;
- если edit-sheet ещё открыт на момент обработки результата, кнопка Play внутри него должна сразу ссылаться на новый URL.

### 4. Immediate playback after generation/regeneration

После успешной генерации или регенерации аудио:

- новое аудио должно запускаться автоматически;
- проигрываться должен именно новый `audioUrl`, полученный от сервера;
- не требуется модальное окно, toast-confirmation, checkbox или второе нажатие от пользователя;
- если браузер заблокировал `Audio.play()`, ошибка должна быть обработана мягко, без падения UI, а кнопка Play должна остаться доступной для ручного повторного запуска.

### 5. Save flow behavior

#### Save Changes

- если primary text не изменился, сохранение не обязано регенерировать аудио;
- если primary text изменился и язык ноты поддерживает TTS, сервер генерирует новое аудио;
- при успешной генерации клиент сразу обновляет local audio state и авто-проигрывает новый файл.

#### Save & Regenerate Audio

- всегда форсирует регенерацию аудио для note с поддерживаемым TTS-языком при валидной форме;
- после ответа сервера клиент сразу подменяет текущий `audioUrl`;
- новое аудио сразу автоматически проигрывается.
- если аудио не удалось сгенерировать или закешировать, форма показывает понятную ошибку и не делает вид, что regenerate прошёл успешно.

## State Requirements

На клиенте должна существовать локальная модель состояния для списка нотов на странице колоды, достаточная чтобы:

- хранить `noteId -> audioUrl`;
- обновлять конкретную ноту после успешного сохранения;
- пересчитывать видимость Play-кнопки без server refresh;
- не терять актуальный `audioUrl`, пока `router.refresh()` доезжает или если он вообще не нужен для этого сценария.

`router.refresh()` допускается как фоновая синхронизация, но не как единственный механизм для появления Play-кнопки и доступа к новому аудио.

## Suggested Implementation Direction

- Вынести список нотов страницы колоды в client component, который получает initial notes и initial audio map.
- Для проигрывания использовать единый helper/hook, создающий `new Audio(audioUrl).play().catch(...)`.
- Передавать в `NoteEditSheet` текущий `audioUrl` и callback `onSaveSuccess`, который возвращает:
  - обновлённые fields,
  - новый `audioUrl` (если он был пересоздан).
- После `onSaveSuccess`:
  - обновлять note row локально;
  - обновлять `audioUrl` в открытом edit flow;
  - запускать autoplay нового аудио на клиенте.

## Acceptance Criteria

- [ ] На `/deck/[id]` у ноты с аудио видна кнопка Play в строке списка.
- [ ] В `NoteEditSheet` у ноты с аудио видна кнопка Play.
- [ ] Нажатие Play в строке ноты воспроизводит текущее аудио.
- [ ] Нажатие Play в edit mode воспроизводит текущее аудио.
- [ ] После изменения слова и сохранения новая кнопка/новый URL доступны сразу, без перезагрузки страницы.
- [ ] После `Save & Regenerate Audio` новый URL также становится доступным сразу, без перезагрузки страницы.
- [ ] После успешной генерации/регенерации новое аудио автоматически проигрывается без дополнительного подтверждения пользователя.
- [ ] Если autoplay браузером заблокирован, UI не падает, а ручная кнопка Play остаётся доступной.
- [ ] Фильтр `Без аудио` на странице колоды обновляется в соответствии с локальным состоянием, не требуя ручного reload.
- [ ] Если генерация/регенерация аудио падает, пользователь видит явную ошибку.
