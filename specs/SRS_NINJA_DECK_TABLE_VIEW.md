# SRS Ninja — Feature: Deck Table View and Focused Manual Review

## Context

Страница колоды используется не только для перехода в обычный review, но и как рабочий экран для точечной навигации по словам, фильтрации по теме и быстрых операций над нотами. Карточечный список плохо подходит для такого сценария: он медленно сканируется, занимает слишком много вертикального места и мешает быстро собирать поднабор для ручной тренировки.

## Goal

Переделать страницу колоды в full-width табличный экран, который:

- быстро сканируется на desktop;
- не ограничен узким `max-width` контейнером;
- позволяет фильтровать ноты по тегам и агрегированному FSRS-статусу;
- позволяет фильтровать ноты по наличию аудио;
- позволяет запускать тренировку только по текущему отфильтрованному набору;
- сохраняет быстрые действия над нотой прямо в таблице;
- сохраняет доступ к аудио и массовой генерации аудио.

## Page Layout

Страница колоды:

- использует full-width layout;
- не ограничивается `max-w-*` контейнером;
- допускает горизонтальный scroll таблицы при нехватке ширины;
- сохраняет существующие deck-level действия сверху страницы.

## Table Structure

Таблица отображает одну строку на одну ноту.

Обязательные колонки:

1. `Audio`
   - если у ноты есть аудио, показывается Play-кнопка;
   - если аудио нет, ячейка остаётся пустой.

2. `Word`
   - показывает основной текст ноты;
   - при наличии перевода он может отображаться вторичной строкой.

3. `Status`
   - показывает цветной чип агрегированного FSRS-состояния ноты;
   - рядом показывает процентную оценку того, насколько слово уже закрепилось, если для этого достаточно данных;
   - не показывает отдельную детализацию по `R / P` карточкам.

4. `Actions`
   - иконочные кнопки в одну строку;
   - обязательные действия:
     - `Preview`
     - `Edit`
     - `Delete`
   - каждая кнопка имеет tooltip или понятный `title`/`aria-label`.

Отдельная колонка `Tags` не нужна: теги используются для фильтрации, а не для постоянного вывода в таблице.

## FSRS Aggregation Rules

Статус ноты считается агрегированно по всем её карточкам.

Приоритет состояний:

1. `relearning`
2. `learning`
3. `new`
4. `review`

То есть если хотя бы одна карточка ещё не дошла до стабильного `review`, строка не должна выглядеть как полностью выученная.

## Memory Percentage

В колонке `Status` рядом с FSRS-чипом показывается числовой процент.

Требования:

- это не строгая математическая вероятность из FSRS, а UI-эвристика;
- процент нужен для быстрого сканирования списка;
- он должен быть стабилен и консистентен для одинаковых входных данных;
- если достаточных данных нет, интерфейс может не показывать процент вовсе.

## Filters

Над таблицей должна быть одна строка компактных фильтров.

### Tag Filter

- строится из реально существующих тегов текущей колоды;
- оформляется как dropdown с multi-select;
- поддерживает поиск по тегам прямо внутри dropdown;
- поиск должен позволять поставить фокус в input и вводить текст без конфликтов с навигацией меню;
- внутри поля поиска есть кнопка быстрого сброса;
- быстрый сброс очищает и поисковую строку, и выбранный tag-filter;
- при выборе нескольких тегов остаются только ноты, содержащие все выбранные теги.

Для English deck:

- технический префикс `ENGLISH::` не показывается в label фильтра;
- фильтрация при этом всё равно работает по исходному полному тегу.

### FSRS Filter

- оформляется как dropdown с multi-select;
- работает по агрегированному FSRS-состоянию ноты;
- поддерживает выбор нескольких состояний одновременно.

### Filter Row Behavior

- фильтры не должны заметно прыгать по ширине при выборе значений;
- триггеры должны иметь стабильную ширину;
- длинные summary-значения должны обрезаться, а не раздвигать layout.

### Audio Filter

- оформляется как compact dropdown;
- поддерживает ровно три состояния:
  - `All notes`
  - `With audio`
  - `Without audio`
- определяет наличие аудио по `audio_cache` для `field_key = 'expression'`;
- влияет и на таблицу, и на `Train shown cards`;
- сериализуется в URL вместе с остальными deck filters.

## Manual Review of Visible Rows

На странице колоды должна быть кнопка:

- `Train shown cards`

Поведение:

- кнопка строит ручную review-сессию только по нотам, которые сейчас видимы после фильтрации;
- в review попадают все карточки этих нотов;
- режим не должен ограничиваться только `due`-очередью;
- URL-фильтры должны переиспользоваться сервером для пересборки того же набора.

## Card Preview

У каждой строки есть один preview-control.

Поведение preview:

- открывается dialog без ухода со страницы колоды;
- внутри используется существующий компонент `Flashcard`, а не отдельная дублирующая разметка;
- внутри dialog есть control для переключения `лицо / оборот`;
- preview не пишет review history;
- preview может проигрывать аудио, если оно есть у ноты.

## Audio Generation

Если колода English, на странице остаётся кнопка массовой генерации аудио.

Требования:

- кнопка использует иконки, а не emoji;
- кнопка показывает, сколько нот будет обработано;
- tooltip поясняет, сколько именно нот попадёт в генерацию;
- если генерировать нечего, кнопка disabled, но её label не исчезает;
- генерация должна учитывать текущие фильтры страницы, а не всю колоду;
- после генерации Play-кнопки должны появляться без reload страницы.
- если generation/regeneration падает, пользователь должен увидеть явную ошибку, а не silent no-op.

## Existing Features To Preserve

- обычный deck review;
- создание нового нота;
- редактирование ноты через `NoteEditSheet`;
- удаление ноты;
- мгновенное локальное обновление списка после edit/delete/audio generation.

## Storybook Requirement

Новый UI должен быть продемонстрирован в Storybook по атомарным слоям, а не только целиком страницей.

Минимально должны быть отдельные stories для:

- статусного FSRS-бейджа;
- filter bar;
- preview dialog;
- кнопки генерации аудио.

## Acceptance Criteria

- [ ] Страница колоды использует full-width layout без узкого контейнера.
- [ ] Ноты отображаются в таблице.
- [ ] В таблице есть колонки `Audio`, `Word`, `Status`, `Actions`.
- [ ] Колонка `Status` показывает цветной FSRS-чип и процент, без `R / P` детализации.
- [ ] У таблицы нет отдельной колонки `Tags`.
- [ ] У строки есть один preview-action, а переключение лицо/оборот делается внутри dialog.
- [ ] Preview использует существующий `Flashcard`.
- [ ] Фильтр по тегам сделан как dropdown multi-select с рабочим поиском и быстрым сбросом.
- [ ] Фильтр по FSRS-состояниям сделан как dropdown multi-select.
- [ ] Есть фильтр `All notes / With audio / Without audio`.
- [ ] Для English deck теги в UI не показывают префикс `ENGLISH::`.
- [ ] Кнопка `Train shown cards` тренирует только текущий отфильтрованный набор.
- [ ] Кнопка `Generate Audio` работает по текущему фильтру и показывает счётчик.
- [ ] Если генерировать нечего, `Generate Audio` остаётся видимой, но disabled.
- [ ] Ошибки audio generation/regeneration не теряются молча и показываются пользователю.
- [ ] Новый UI показан в Storybook атомарными stories.
